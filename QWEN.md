# QWEN Project Configuration

## О проекте

VaseForge — веб-приложение для создания 3D-моделей ваз в формате STL для последующей 3D-печати. Пользователь задаёт параметры вазы (высота, радиусы, форма сечения) и формулы, на основе которых система автоматически генерирует геометрию. Результат экспортируется в STL для печати.

## Стек технологий

| Категория | Технология | Версия | Примечания |
|-----------|-----------|--------|------------|
| Язык | TypeScript | — | Основной язык |
| Фронтенд | React + Vite | — | UI-фреймворк |
| 3D-рендеринг | Three.js / React Three Fiber | — | Визуализация в браузере |
| Генерация геометрии | Собственный парсер формул + Three.js BufferGeometry | — | Ядро проекта |
| Экспорт | stl.js / custom STL writer | — | Генерация .stl файлов |
| Состояние | Zustand / Jotai | — | State management |
| Парсер формул | math.js / custom | — | Вычисление параметров по формулам |

## Правила работы с документами

### Шаблонизация
- Все новые документы создаются на основе шаблонов из папки `templates/`
- Шаблонные папки `docs/*/TEMPLATE.md` содержат пример структуры
- Заполните все обязательные поля (ID, статус, описание)

### Статусы
- **Черновик** — документ в разработке, не утверждён
- **На рассмотрении** — готов к ревью
- **Согласовано** — утверждён, готов к использованию
- **Отклонено** — не принято,归档

### Связи между документами
- BR → FR → US → TASK (каскадная декомпозиция)
- FR ↔ UC (покрытие функциональности)
- Все документы должны содержать секцию «Связанные элементы»

## Рабочий процесс (Workflow)

### 1. Планирование
1. Сформулировать бизнес-требования (BR)
2. Декомпозировать в функциональные требования (FR)
3. Определить Use Cases (UC) для ключевых сценариев
4. Сформировать User Stories (US)
5. Разбить US на задачи (TASK)

### 2. Анализ
1. Изучить требования и контекст
2. Выявить нефункциональные требования (NFR)
3. Зафиксировать архитектурные решения (ADR)
4. Определить зависимости и риски

### 3. Разработка
1. Взять задачу из бэклога
2. Изучить связанные US/UC/FR
3. Реализовать и протестировать
4. Обновить статус задачи

### 4. Проверка
1. Критерии приёмки из US должны быть выполнены
2. Документация должна соответствовать реализацию
3. Связи между документами — консистентны
