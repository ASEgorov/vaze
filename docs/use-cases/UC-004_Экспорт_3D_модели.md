## Основная информация

| Параметр | Значение |
|----------|----------|
| **Название** | Экспорт 3D-модели в STL |
| **ID** | UC-004 |
| **Актер** | Пользователь |
| **Цель** | Генерация и скачивание STL-файла из 3D-модели вазы |
| **Предусловия** | Пользователь满意 моделью, геометрия валидна (UC-002 завершён) |
| **Постусловия** | Пользователь скачал STL-файл, готовый к загрузке в слайсер |

## Основной поток (Basic Flow)

| Шаг | Актер | Система | Комментарий |
|-----|-------|---------|-------------|
| 1 | Нажимает кнопку «Экспорт STL» | Запрашивает формат экспорта (бинарный / ASCII) | Диалог или переключатель |
| 2 | Выбирает формат (бинарный по умолчанию) | Применяет выбор | Бинарный — меньший размер файла |
| 3 | Вводит имя файла (опционально) | Применяет имя | По умолчанию `vase_export.stl` |
| 4 | — | Проверяет валидность геометрии (нет разрывов, NaN) | Проверка треугольников |
| 5 | — | Применяет текущий масштаб модели | Из настроек печати |
| 6 | — | Конвертирует BufferGeometry в плоский список треугольников | [x1,y1,z1, x2,y2,z2, x3,y3,z3] |
| 7 | — | Формирует STL-файл: заголовок + массив треугольников | Бинарный: 80B + N*50B |
| 8 | — | Создаёт Blob и ссылку для скачивания | URL.createObjectURL |
| 9 | — | Инициирует скачивание файла в браузер пользователя | fileSaver / download trigger |
| 10 | — | Показывает уведомление: `Файл vase_export.stl скачан` | Toast-уведомление |

## Альтернативные потоки (Alternative Flows)

### AF-1: Экспорт ASCII STL
- **Шаг:** 2
- **Условие:** Пользователь выбрал ASCII формат
- **Действие:** Система генерирует текстовый STL по формату `facet normal nx ny nz / outer loop / vertex x y z / endloop / endfacet`
- **Возврат к:** Шаг 7 (ASCII-формат)

### AF-2: Применение пользовательского масштаба
- **Шаг:** 5
- **Условие:** Пользователь задал масштаб (например, 80%)
- **Действие:** Система умножает все координаты вершин на scaleFactor перед генерацией STL
- **Возврат к:** Шаг 6

### AF-3: Экспорт с изменённой ориентацией
- **Шаг:** 5
- **Условие:** Пользователь выбрал ориентацию «На бок» (поворот на 90°)
- **Действие:** Система применяет матрицу вращения к вершинам перед экспортом
- **Возврат к:** Шаг 6

### AF-4: Экспорт с выбором количества сегментов
- **Шаг:** 6
- **Условие:** Пользователь выбрал количество сегментов для экспорта (не обязательно полное)
- **Действие:** Система генерирует геометрию с указанным количеством сегментов перед конвертацией
- **Возврат к:** Шаг 6

## Потоки ошибок (Error Flows)

### EF-1: Геометрия невалидна (разрывы)
- **Шаг:** 4
- **Ошибка:** Обнаружены разрывы в треугольной сетке
- **Обработка:** Показать: `Невозможно экспортировать: модель содержит разрывы. Проверьте формулу.`
- **Возврат к:** Шаг 1 (возврат к редактированию)

### EF-2: Пустая геометрия
- **Шаг:** 4
- **Ошибка:** Нет геометрии для экспорта (формула не введена)
- **Обработка:** Показать: `Сначала создайте модель.`
- **Возврат к:** — (перенаправление в редактор)

### EF-3: Файл слишком большой (>100MB)
- **Шаг:** 7
- **Ошибка:** Сформированный STL превышает лимит
- **Обработка:** Показать: `Файл слишком большой. Уменьшите детализацию или масштаб.`
- **Возврат к:** Шаг 1

## Бизнес-правила

| ID | Правило | Описание |
|----|---------|----------|
| BP-01 | Формат STL | Поддержка бинарного и ASCII STL |
| BP-02 | Единицы | STL не хранит единицы; все размеры в мм |
| BP-03 | Точность | 3 знака после запятой для координат (0.001 мм) |
| BP-04 | Нормали | Вычисляются автоматически для каждого треугольника |
| BP-05 | Совместимость | Файл должен открываться в Cura, PrusaSlicer, Simplify3D |

## Данные и интерфейсы

### Входные данные
| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| geometry | BufferGeometry | Да | 3D-геометрия от UC-002 |
| format | string | Нет | `binary` или `ascii`, по умолчанию `binary` |
| filename | string | Нет | Имя файла, по умолчанию `vase_export.stl` |
| scale | number | Нет | Масштабный множитель, по умолчанию 1.0 |
| orientation | string | Нет | Ориентация: `base`, `side`, `custom` |

### Выходные данные
| Параметр | Тип | Описание |
|----------|-----|----------|
| file | Blob | STL-файл для скачивания |
| fileSize | number | Размер файла в байтах |
| triangleCount | number | Количество треугольников в файле |
| downloadUrl | string | URL для триггера скачивания |
