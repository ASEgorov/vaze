## Основная информация

| Параметр | Значение |
|----------|----------|
| **Название** | Управление проектами |
| **ID** | UC-005 |
| **Актер** | Пользователь |
| **Цель** | Сохранение, загрузка, удаление и управление набором проектов |
| **Предусловия** | Пользователь создал хотя бы один проект (или хочет загрузить) |
| **Постусловия** | Проекты доступны в списке, выбран для редактирования или удалён |

## Основной поток (Basic Flow)

| Шаг | Актер | Система | Комментарий |
|-----|-------|---------|-------------|
| 1 | — | Загружает список сохранённых проектов из хранилища | IndexedDB / localStorage |
| 2 | — | Отображает список проектов с превью | Карточки: имя, дата, мини-превью |
| 3 | Нажимает «Сохранить» | Запрашивает имя проекта | Диалог сохранения |
| 4 | Вводит имя `Красивая ваза` | Валидирует имя (уникальность, длина) | 1–50 символов |
| 5 | — | Сериаализует состояние проекта: формула, параметры, камера, настройки | JSON |
| 6 | — | Сохраняет проект в хранилище с timestamp | { id, name, data, createdAt, updatedAt } |
| 7 | — | Обновляет список проектов (добавляет в начало) | Визуальное подтверждение |
| 8 | Нажимает на проект в списке | — | Выбор проекта |
| 9 | — | Загружает данные проекта и восстанавливает состояние | Все поля редактора |
| 10 | — | Обновляет 3D-превью с загруженной моделью | Связано с UC-003 |
| 11 | Нажимает «Удалить» на проекте | Запрашивает подтверждение удаления | Диалог: «Удалить {name}?» |
| 12 | Подтверждает удаление | Удаляет проект из хранилища | Безвозвратное удаление |
| 13 | — | Обновляет список (проект исчезает) | Визуальное подтверждение |

## Альтернативные потоки (Alternative Flows)

### AF-1: Автосохранение
- **Шаг:** 5
- **Условие:** Прошло 30 секунд без действий пользователя
- **Действие:** Система сохраняет черновик с префиксом `[Черновик]` и автоименем
- **Возврат к:** Шаг 7

### AF-2: Импорт из JSON-файла
- **Шаг:** 3
- **Условие:** Пользователь импортирует JSON-файл проекта
- **Действие:** Система читает файл, парсит JSON, валидирует структуру, создаёт проект
- **Возврат к:** Шаг 5

### AF-3: Загрузка при несохранённых изменениях
- **Шаг:** 8
- **Условие** Пользователь редактирует текущую формулу, но не сохранил изменения
- **Действие:** Система показывает: `Есть несохранённые изменения. Сохранить? Да / Нет / Отмена`
- **Возврат к:** Шаг 3 (сохранить) или Шаг 8 (без сохранения)

### AF-4: Лимит проектов достигнут
- **Шаг:** 6
- **Условие:** В хранилище 50 проектов
- **Действие:** Система удаляет самый старый проект и создаёт новый, или показывает: `Достигнут лимит. Удалите старые проекты.`
- **Возврат к:** Шаг 4

### AF-5: Восстановление повреждённого проекта
- **Шаг:** 9
- **Условие:** Данные проекта повреждены (невалидный JSON)
- **Действие:** Система показывает: `Проект повреждён. Попробуйте восстановить.` и предлагает версию из автосохранения
- **Возврат к:** Шаг 8 (выбор версии)

## Потоки ошибок (Error Flows)

### EF-1: Хранилище заполнено (Quota Exceeded)
- **Шаг:** 6
- **Ошибка:** localStorage исчерпан (обычно 5-10MB)
- **Обработка:** Показать: `Хранилище заполнено. Удалите старые проекты.`
- **Возврат к:** Шаг 1

### EF-2: Ошибка чтения хранилища
- **Шаг:** 1
- **Ошибка:** Хранилище повреждено или недоступно
- **Обработка:** Показать: `Ошибка загрузки проектов. Попробуйте импортировать из резервной копии.`
- **Возврат к:** — (список пуст)

## Бизнес-правила

| ID | Правило | Описание |
|----|---------|----------|
| BP-01 | Лимит проектов | Максимум 50 проектов в хранилище |
| BP-02 | Имя проекта | 1–50 символов, уникальность в рамках сессии |
| BP-03 | Автосохранение | Каждые 30 секунд при наличии изменений |
| BP-04 | Формат данных | JSON-структура: { formula, params, camera, createdAt } |
| BP-05 | Удаление | Безвозвратное, только с подтверждением пользователя |

## Данные и интерфейсы

### Входные данные
| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| projectName | string | Нет | Имя проекта (для сохранения) |
| projectData | object | Да | JSON-данные проекта (формула, параметры, камера) |
| importFile | File | Нет | JSON-файл для импорта |

### Выходные данные
| Параметр | Тип | Описание |
|----------|-----|----------|
| projects | Project[] | Список всех проектов |
| currentProject | Project | Текущий открытый проект |
| savedProject | Project | Сохранённый проект с id и timestamp |
| deletedProjectId | string | ID удалённого проекта |
