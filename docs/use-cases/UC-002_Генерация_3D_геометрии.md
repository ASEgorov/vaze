## Основная информация

| Параметр | Значение |
|----------|----------|
| **Название** | Генерация 3D-геометрии вазы |
| **ID** | UC-002 |
| **Актер** | Пользователь, Система (автоматически после UC-001) |
| **Цель** | Построение треугольной сетки (mesh) 3D-модели вазы по массиву точек сечения |
| **Предусловия** | Парсер сформировал массив точек [(r, h)] — UC-001 завершён успешно |
| **Постусловия** | Система создала 3D-меш (BufferGeometry), готовый к визуализации и экспорту |

## Основной поток (Basic Flow)

| Шаг | Актер | Система | Комментарий |
|-----|-------|---------|-------------|
| 1 | — | Получает массив точек [(r, h)] от парсера (UC-001) | 64–256 точек по умолчанию |
| 2 | — | Создаёт массив 3D-точек (vertices) методом латхе (lathe geometry) | Вращение точки (r, h) вокруг оси Y |
| 3 | — | Генерирует индексы (faces/triangles) для формирования треугольников | 2 triangles per quad |
| 4 | — | Вычисляет нормали для каждой вершины (vertex normals) | Для корректного освещения |
| 5 | — | Вычисляет UV-координаты для текстурирования (опционально) | U: угол, V: высота |
| 6 | — | Создаёт THREE.BufferGeometry (или R3F <BufferGeometry>) | Оптимизированная структура |
| 7 | — | Передаёт геометрию в 3D-сцену для рендеринга | Связано с UC-003 |
| 8 | — | Обновляет bounding box модели (для проверки платформы) | Используется в US-011 |

## Альтернативные потоки (Alternative Flows)

### AF-1: Непрерывное обновление при изменении формулы
- **Шаг:** 1
- **Условие:** Пользователь меняет формулу или параметры
- **Действие:** Система отменяет предыдущую генерацию (cancelAnimationFrame) и запускает генерацию заново с новыми данными
- **Возврат к:** Шаг 1

### AF-2: Геометрия с разрывами (NaN в точках)
- **Шаг:** 2
- **Условие:** Некоторые точки содержат NaN (AF-3 из UC-001)
- **Действие:** Система пропускает проблемные точки, создаёт несколько мешей вместо одного, показывает предупреждение
- **Возврат к:** Шаг 3 (генерация с пропуском)

### AF-3: Очень высокая детализация
- **Шаг:** 6
- **Условие:** Пользователь установил большое количество сегментов (>512)
- **Действие:** Система предупреждает о возможной потере производительности и предлагает оптимизацию
- **Возврат к:** Шаг 6 (с ограничением до 512)

### AF-4: Самопересекающаяся геометрия
- **Шаг:** 3
- **Условие:** Форма вазы приводит к самопересечению (например, r < 0 при某些 h)
- **Действие:** Система фильтрует негативные радиусы, предупреждает пользователя
- **Возврат к:** Шаг 3

## Потоки ошибок (Error Flows)

### EF-1: Пустой массив точек
- **Шаг:** 1
- **Ошибка:** Парсер вернул пустой массив
- **Обработка:** Показать: `Невозможно построить модель. Проверьте формулу.`
- **Возврат к:** Шаг 1

### EF-2: Слишком много точек — OOM
- **Шаг:** 6
- **Ошибка:** Нехватка памяти для создания меша
- **Обработка:** Показать: `Слишком высокая детализация. Уменьшите количество сегментов.`
- **Возврат к:** Шаг 1

## Бизнес-правила

| ID | Правило | Описание |
|----|---------|----------|
| BP-01 | Метод генерации | LatheGeometry — вращение профиля вокруг оси |
| BP-02 | Количество сегментов | По умолчанию 64, диапазон 16–1024 |
| BP-03 | Ось вращения | Ось Y (вертикальная ваза) |
| BP-04 | Ориентация | Основание на уровне h=0 |
| BP-05 | Тип меша | Non-indexed (для простоты экспорта в STL) |

## Данные и интерфейсы

### Входные данные
| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| points | Point2D[] | Да | Массив [{r, h}] от парсера |
| segments | number | Нет | Количество сегментов поворота, по умолчанию 64 |
| scale | number | Нет | Масштабный множитель, по умолчанию 1.0 |

### Выходные данные
| Параметр | Тип | Описание |
|----------|-----|----------|
| geometry | BufferGeometry | Three.js геометрия для рендеринга |
| vertices | Vector3[] | Массив 3D-вершин |
| indices | Uint16Array | Индексы треугольников |
| normals | Float32Array | Нормали вершин |
| boundingBox | Box3 | Ограничивающий параллелепипед модели |
| triangleCount | number | Количество треугольников в меше |
