## Описание

**Как** пользователь приложения VaseForge,  
**я хочу** настроить масштаб модели и её ориентацию перед экспортом в STL,  
**чтобы** модель соответствовала размерам 3D-принтера и оптимизировала расход материала.

## Контекст

Перед печатью пользователь должен убедиться, что модель помещается на платформу принтера и правильно ориентирована. Масштабирование и поворот влияют на качество печати и расход филамента.

## Критерии приёмки (Acceptance Criteria)

### Сценарий 1: Установка масштаба в процентах — Happy Path
- **Дано** пользователь满意 моделью
- **Когда** пользователь вводит `80` в поле «Масштаб»
- **То** 3D-превью обновляется, отображая модель с уменьшенным на 20% размером

### Сценарий 2: Выбор ориентации «На основание» — Happy Path
- **Дано** пользователь в разделе настроек печати
- **Когда** пользователь выбирает ориентацию «На основание»
- **То** модель поворачивается так, что основание смотрит вниз (ось Z)

### Сценарий 3: Выбор ориентации «На бок» — Happy Path
- **Дано** пользователь в разделе настроек печати
- **Когда** пользователь выбирает ориентацию «На бок»
- **То** модель поворачивается на 90° вокруг оси X

### Сценарий 4: Отображение размеров модели — Happy Path
- **Дано** пользователь настроил масштаб и ориентацию
- **Когда** пользователь наводит курсор на блок «Размеры»
- **То** отображаются: `Высота: XX мм × Диаметр: XX мм`

### Сценарий 5: Проверка размера платформы — Warning
- **Дано** платформа принтера пользователя 220×220 мм
- **Когда** модель после масштабирования имеет диаметр 250 мм
- **То** система показывает предупреждение: `Модель не помещается на платформу 220×220 мм. Уменьшите масштаб.`

## Технические заметки

- Масштаб: 10–500%, шаг 1%
- Ориентации: base (0°), side (90°), custom (вращение по осям X/Y/Z)
- Проверка платформы: предустановленные размеры (220×220, 300×300, custom)
- Размеры модели: bounding box из 3D-геометрии
