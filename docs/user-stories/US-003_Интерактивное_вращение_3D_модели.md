## Описание

**Как** пользователь приложения VaseForge,  
**я хочу** вращать, масштабировать и перемещать 3D-модель вазы мышью или тачпадом,  
**чтобы** рассмотреть форму со всех сторон и оценить детали.

## Контекст

Статичное превью недостаточно — пользователь должен иметь возможность осмотреть модель полностью, как в профессиональных 3D-редакторах. Это стандартное поведение для любой 3D-сцены.

## Критерии приёмки (Acceptance Criteria)

### Сценарий 1: Вращение модели — Happy Path
- **Дано** на экране отображается 3D-модель вазы
- **Когда** пользователь зажимает левую кнопку мыши и двигает курсор
- **То** камера вращается вокруг модели, изменяя угол обзора

### Сценарий 2: Масштабирование колёсиком — Happy Path
- **Дано** курсор находится в области 3D-сцены
- **Когда** пользователь вращает колёсико мыши
- **То** модель масштабируется (zoom in / zoom out) с плавной анимацией

### Сценарий 3: Панорамирование — Happy Path
- **Дано** модель масштабируется больше, чем область просмотра
- **Когда** пользователь зажимает правую кнопку мыши и двигает
- **То** камера перемещается параллельно (pan)

### Сценарий 4: Сброс камеры — Happy Path
- **Дано** пользователь изменил позицию камеры
- **Когда** пользователь нажимает кнопку «Сброс камеры»
- **То** камера возвращается к начальной позиции (вид спереди, стандартный зум)

### Сценарий 5: Подсветка выбранной грани — Interactive
- **Дано** пользователь наводит курсор на модель
- **Когда** курсор находится над гранью треугольника
- **То** грань подсвечивается (hover-эффект)

## Технические заметки

- OrbitControls из Three.js или аналог для React Three Fiber
- Поддержка тач-жестов для мобильных устройств: pinch-to-zoom, two-finger rotate
- Оптимизация: frustum culling, LOD для сложных моделей
- Производительность: target 60 FPS для моделей до 100K полигонов
