## Описание

**Как** пользователь приложения VaseForge,  
**я хочу** ввести формулу для расчёта радиуса сечения вазы в зависимости от высоты,  
**чтобы** система могла построить геометрию вазы с заданной формой.

## Контекст

Это основная точка входа в рабочий процесс. Пользователь описывает форму вазы через математическое выражение r = f(h), где h — высота (от 0 до заданной максимальной), r — радиус сечения на данной высоте. Формула — ядро продукта.

## Критерии приёмки (Acceptance Criteria)

### Сценарий 1: Ввод простой линейной формулы — Happy Path
- **Дано** пользователь открыл редактор формул
- **Когда** пользователь вводит `r = h * 0.5`
- **То** система принимает формулу как валидную и отображает превью конической вазы

### Сценарий 2: Ввод формулы с тригонометрией — Happy Path
- **Дано** пользователь открыл редактор формул
- **Когда** пользователь вводит `r = 5 + 3 * sin(h * 0.1)`
- **То** система принимает формулу и отображает волнообразную вазу

### Сценарий 3: Ввод формулы с некорректным синтаксисом — Error Case
- **Дано** пользователь открыл редактор формул
- **Когда** пользователь вводит `r = 5 + sin(h *` (незакрытая скобка)
- **То** система подсвечивает ошибку и показывает сообщение: `Ожидалась закрывающая скобка`

### Сценарий 4: Ввод формулы с неизвестной функцией — Error Case
- **Дано** пользователь открыл редактор формул
- **Когда** пользователь вводит `r = exp(h)`
- **То** система показывает подсказку: `Функция exp не найдена. Доступные функции: sin, cos, tan, pow, abs, sqrt`

## Технические заметки

- Переменная высоты: `h` (диапазон от 0 до `maxHeight`, мм)
- Переменная угла: `θ` или `theta` (опционально, для спиральных ваз)
- Доступные операции: +, -, *, /, ^ (степень)
- Доступные функции: sin, cos, tan, pow, abs, sqrt, log, pi, e
- Парсинг — через math.js или собственный рекурсивный спуск
- Формула должна вычислять массив точек для генерации поверхности
