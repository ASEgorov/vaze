## Описание

**Как** пользователь приложения VaseForge,  
**я хочу** загружать ранее сохранённый проект,  
**чтобы** продолжить работу с места остановки или использовать чужой дизайн как основу.

## Контекст

После сохранения проекты становятся доступными в списке. Пользователь может выбрать любой проект, загрузить его и продолжить редактирование. Также важно уметь делиться проектами с другими пользователями.

## Критерии приёмки (Acceptance Criteria)

### Сценарий 1: Загрузка из списка проектов — Happy Path
- **Дано** пользователь ранее сохранил проект `Красивая ваза`
- **Когда** пользователь открывает «Мои проекты» и нажимает на `Красивая ваза`
- **То** загружается формула, параметры, настройки камеры и отображается 3D-модель

### Сценарий 2: Загрузка при наличии несохранённых изменений — Warning
- **Дано** пользователь редактирует текущую формулу и не сохранил изменения
- **Когда** пользователь нажимает «Загрузить другой проект»
- **То** система показывает: `Есть несохранённые изменения. Сохранить? Да / Нет / Отмена`

### Сценарий 3: Импорт из файла — Happy Path
- **Дано** пользователь получил JSON-файл проекта от другого пользователя
- **Когда** пользователь нажимает «Импорт» и выбирает файл
- **То** система загружает проект из JSON-файла

### Сценарий 4: Загрузка несуществующего проекта — Error Case
- **Дано** проект был удалён или хранилище повреждено
- **Когда** пользователь нажимает на проект
- **То** система показывает: `Проект не найден. Возможно, он был удалён.`

### Сценарий 5: Удаление проекта — Happy Path
- **Дано** пользователь открыл список «Мои проекты»
- **Когда** пользователь нажимает «Удалить» на проекте и подтверждает
- **То** проект удаляется из хранилища и списка

## Технические заметки

- Список проектов: `{ id, name, createdAt, updatedAt, previewData }`
- Полный проект: `{ formula, params, camera, createdAt }`
- Импорт/экспорт: JSON-файл для обмена между пользователями
- Валидация: проверка структуры JSON при импорте
