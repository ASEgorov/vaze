## Описание

**Как** пользователь приложения VaseForge,  
**я хочу** экспортировать готовую 3D-модель в STL-файл,  
**чтобы** открыть его в слайсере и отправить на 3D-печать.

## Контекст

Экспорт в STL — финальный шаг рабочего процесса. После создания и одобрения модели пользователь должен одним действием получить файл, совместимый со всеми популярными слайсерами (Cura, PrusaSlicer, Simplify3D).

## Критерии приёмки (Acceptance Criteria)

### Сценарий 1: Экспорт бинарного STL — Happy Path
- **Дано** пользователь满意 моделью на экране
- **Когда** пользователь нажимает кнопку «Экспорт STL»
- **То** браузер скачивает файл `vase_export.stl` в бинарном формате

### Сценарий 2: Экспорт ASCII STL — Happy Path
- **Дано** пользователь满意 моделью
- **Когда** пользователь выбирает формат ASCII в диалоге экспорта и нажимает «Экспорт»
- **То** браузер скачивает файл `vase_export.stl` в текстовом формате

### Сценарий 3: Выбор имени файла — Happy Path
- **Дано** пользователь满意 моделью
- **Когда** пользователь вводит имя `my_vase` в поле и нажимает «Экспорт»
- **То** скачивается файл `my_vase.stl`

### Сценарий 4: Экспорт с невалидной геометрией — Error Case
- **Дано** формула создаёт самопересекающуюся или некорректную геометрию
- **Когда** пользователь нажимает «Экспорт STL»
- **То** система показывает ошибку: `Невозможно экспортировать: геометрия содержит разрывы. Проверьте формулу.`

### Сценарий 5: Экспорт без заданной формулы — Error Case
- **Дано** пользователь не вводил формулу и параметры
- **Когда** пользователь нажимает «Экспорт STL»
- **То** система показывает: `Сначала создайте модель, введя формулу и параметры.`

## Технические заметки

- Бинарный STL: 80 байт заголовок + 4 байта количество треугольников + 50 байт на треугольник
- ASCII STL: формат `facet normal nx ny nz / outer loop / vertex x y z / endloop / endfacet`
- Генерация поверхности: латхе (lathe) через массив точек из формулы
- Оптимизация: количество сегментов по умолчанию 64, настраиваемое пользователем
