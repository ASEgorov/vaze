## Описание

**Как** пользователь приложения VaseForge,  
**я хочу** сохранять текущий проект с формулами и параметрами,  
**чтобы** продолжить работу позже или поделиться с другими.

## Контекст

Пользователь может работать над проектом длительное время, прерываться, и возвращаться позже. Сохранение — ключевой функционал для удобства. Также полезно для обмена дизайнами между пользователями.

## Критерии приёмки (Acceptance Criteria)

### Сценарий 1: Сохранение проекта с именем — Happy Path
- **Дано** пользователь ввёл формулу и параметры, построил модель
- **Когда** пользователь нажимает «Сохранить», вводит имя `Красивая ваза` и подтверждает
- **То** проект сохраняется в локальное хранилище (IndexedDB / localStorage)

### Сценарий 2: Сохранение с автоименем — Happy Path
- **Дано** пользователь не ввёл имя при сохранении
- **Когда** пользователь нажимает «Сохранить» без имени
- **То** система генерирует имя по шаблону `Ваза_YYYY-MM-DD_HH:mm`

### Сценарий 3: Сохранение без изменений — Happy Path
- **Дано** пользователь не вносил изменений с последнего сохранения
- **Когда** пользователь нажимает «Сохранить»
- **То** система показывает: `Нет несохранённых изменений`

### Сценарий 4: Сохранение без формулы — Error Case
- **Дано** пользователь не вводил формулу
- **Когда** пользователь нажимает «Сохранить»
- **То** система показывает: `Нельзя сохранить: введите формулу и параметры модели.`

### Сценарий 5: Автосохранение — Happy Path
- **Дано** пользователь работает с формулой более 30 секунд
- **Когда** проходит 30 секунд без действий пользователя
- **То** система сохраняет черновик с пометкой `[Черновик]` и показывает уведомление

## Технические заметки

- Хранилище: IndexedDB (для больших данных) или localStorage (для простых данных)
- Формат сохранения: JSON со структурой `{ formula, params, camera, timestamp }`
- Автосохранение: debounce с задержкой 30 сек
- Лимит: 50 проектов на локальное хранилище
