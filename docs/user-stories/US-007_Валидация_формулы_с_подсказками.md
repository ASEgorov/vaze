## Описание

**Как** пользователь приложения VaseForge,  
**я хочу** получать мгновенную валидацию введённой формулы с понятными подсказками об ошибках,  
**чтобы** быстро исправить проблему и не теряться в неясных сообщениях.

## Контекст

Пользователи могут иметь разный уровень математической подготовки. Невалидная формула не должна приводить к падению — система должна объяснить, что не так, и подсказать, как исправить.

## Критерии приёмки (Acceptance Criteria)

### Сценарий 1: Ошибка синтаксиса — незакрытая скобка — Error Case
- **Дано** пользователь вводит формулу `r = pow(h, 2) + sin(`
- **Когда** пользователь отпускает клавишу
- **То** система подсвечивает строку красным и показывает: `Ожидалось выражение после '('`

### Сценарий 2: Неизвестная функция — Error Case
- **Дано** пользователь вводит формулу `r = exp(h)`
- **Когда** пользователь отпускает клавишу
- **То** система показывает: `Функция 'exp' не найдена. Доступные: sin, cos, tan, pow, abs, sqrt, log`

### Сценарий 3: Ошибка вычисления — деление на ноль — Error Case
- **Дано** пользователь вводит валидную формулу `r = 1 / (h - 50)`
- **Когда** система вычисляет значение при h=50
- **То** система показывает предупреждение: `Деление на ноль при h=50. Модель может иметь разрыв.`

### Сценарий 4: Автодополнение функций — Happy Path
- **Дано** пользователь начинает вводить `si`
- **Когда** пользователь печатает эти символы
- **То** система показывает подсказку автодополнения: `sin(...)`

### Сценарий 5: Валидная формула — Happy Path
- **Дано** пользователь вводит `r = 5 + 2 * cos(h * 0.05)`
- **Когда** пользователь отпускает клавишу
- **То** строка подсвечивается зелёным, ошибка не отображается

## Технические заметки

- Валидация в реальном времени (onKeyPress / onInput с debounce ~200ms)
- Парсер: math.js с кастомными функциями или собственный парсер
- Словарь ошибок: маппинг кодов ошибок на читаемые сообщения
- Автодополнение: список доступных переменных, функций, операторов
