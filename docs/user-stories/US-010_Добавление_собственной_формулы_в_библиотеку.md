## Описание

**Как** пользователь приложения VaseForge,  
**я хочу** добавить свою формулу в библиотеку,  
**чтобы** поделиться ею с другими пользователями или использовать повторно.

## Контекст

Пользователи, создавшие удачные формулы, захотят сохранить их в библиотеку для повторного использования или обмена с сообществом. Это также помогает строить коллекцию пользовательских дизайнов.

## Критерии приёмки (Acceptance Criteria)

### Сценарий 1: Сохранение в библиотеку — Happy Path
- **Дано** пользователь создал и满意 формулу в редакторе
- **Когда** пользователь нажимает «Сохранить в библиотеку», вводит имя и категорию
- **То** формула сохраняется в библиотеку и появляется в каталоге

### Сценарий 2: Сохранение без категории — Happy Path
- **Дано** пользователь сохраняет формулу
- **Когда** пользователь не выбрал категорию
- **То** система использует категорию «Другое» по умолчанию и показывает подсказку

### Сценарий 3: Сохранение дубликата — Warning
- **Дано** формула с таким именем уже существует в библиотеке
- **Когда** пользователь нажимает «Сохранить»
- **То** система показывает: `Формула с именем '{name}' уже существует. Заменить? Да / Нет`

### Сценарий 4: Сохранение невалидной формулы — Error Case
- **Дано** пользователь не ввёл формулу или она содержит ошибки
- **Когда** пользователь нажимает «Сохранить в библиотеку»
- **То** система показывает: `Нельзя сохранить: валидная формула обязательна.`

### Сценарий 5: Редактирование собственной формулы в библиотеке — Happy Path
- **Дано** пользователь добавил формулу в библиотеку
- **Когда** пользователь нажимает «Редактировать» на своей формуле
- **То** формула загружается в редактор для изменения

## Технические заметки

- Библиотека формул: `{ id, name, category, formula, params, author, createdAt, isPublic }`
- Приватные vs публичные формулы (если будет онлайн-разделение)
- Валидация имени: 1–50 символов, уникальность в рамках категории
