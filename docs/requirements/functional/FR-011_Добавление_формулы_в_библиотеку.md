## Описание

Система позволяет пользователю сохранить текущую формулу из редактора в библиотеку формул с именем, категорией и опциональным описанием.

## Входящие данные

| Параметр | Тип | Описание | Обязательный |
|----------|-----|----------|--------------|
| formulaText | string | Текстовая формула | Да |
| name | string | Имя формулы в библиотеке | Да |
| category | string | Категория формулы | Нет, по умолчанию «Другое» |
| description | string | Описание формулы | Нет |
| params | object | Параметры формулы (maxHeight, segments, scale) | Нет |

## Выходные данные

| Параметр | Тип | Описание |
|----------|-----|----------|
| libraryFormulaId | string | ID добавленной формулы в библиотеке |
| name | string | Имя формулы |
| category | string | Категория |
| isSuccess | boolean | Результат добавления |
| errorMessage | string | Сообщение об ошибке |

## Логика обработки

1. Валидация формулы:
   - Вызвать FR-001 для проверки формулы
   - Если невалидна: вернуть ошибку
2. Валидация имени:
   - Длина: 1–50 символов
   - Допустимые символы: буквы, цифры, пробелы, дефис, подчёркивание
   - Проверка на пустоту
3. Проверка уникальности имени в библиотеке:
   - Загрузить список формул библиотеки (локальных + предустановленных)
   - Если имя существует:
     - Если пользовательская формула того же пользователя: показать `Заменить? Да / Нет`
     - Если чужая формула: показать `Формула с таким именем уже существует. Выберите другое имя.`
4. Проверка категории:
   - Если категория не из whitelist: установить `category = 'Другое'`
   - Если категория не указана: установить `category = 'Другое'`
5. Сбор данных формулы:
   ```
   {
     id: generateUUID(),
     name: name,
     category: category,
     formula: { text: formulaText, ...params },
     description: description || '',
     author: 'user',
     isPublic: false,
     createdAt: now(),
     updatedAt: now()
   }
   ```
6. Добавление в библиотеку:
   - Загрузить список пользовательских формул
   - Добавить новую формулу в список
   - Сохранить обратно в хранилище
7. Генерация превью:
   - Вычислить мини-геометрию (segments = 16)
   - Сохранить превью в кэш
8. Обновить UI каталога библиотеки:
   - Добавить карточку новой формулы
   - Обновить счётчик формул
9. Показ уведомления: `Формула '{name}' добавлена в библиотеку`
10. Возврат результатов

## Обработка ошибок

| Код ошибки | Ситуация | Действие системы | Сообщение пользователю |
|------------|----------|------------------|------------------------|
| ERR-911 | Невалидная формула | На шаге 1 | `Нельзя добавить: формула содержит ошибки.` |
| ERR-912 | Некорректное имя | На шаге 2 | `Имя должно быть от 1 до 50 символов (буквы, цифры, пробелы, -_)` |
| ERR-913 | Имя уже занято | На шаге 3 | `Формула с именем '{name}' уже существует.` |
| ERR-914 | Хранилище заполнено | На шаге 6 | `Хранилище заполнено.` |
