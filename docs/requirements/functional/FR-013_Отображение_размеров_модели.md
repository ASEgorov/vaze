## Описание

Система вычисляет и отображает геометрические характеристики 3D-модели в реальном времени: высоту, диаметр, объём, площадь поверхности и количество треугольников.

## Входящие данные

| Параметр | Тип | Описание | Обязательный |
|----------|-----|----------|--------------|
| geometry | BufferGeometry | 3D-геометрия от FR-003 | Да |
| currentScale | number | Текущий масштаб модели | Да |
| maxHeight | number | Максимальная высота (мм) | Да |

## Выходные данные

| Параметр | Тип | Описание |
|----------|-----|----------|
| height | number | Высота модели в мм |
| diameter | number | Максимальный диаметр в мм |
| radius | number | Максимальный радиус в мм |
| volume | number | Объём в мм³ |
| surfaceArea | number | Площадь поверхности в мм² |
| triangleCount | number | Количество треугольников |
| vertexCount | number | Количество вершин |

## Логика обработки

1. Расчёт bounding box из geometry:
   - Вычислить min/max по каждой оси из vertices
   - `height = maxY - minY`
   - `diameter = max(maxX - minX, maxZ - minZ)`
   - `radius = diameter / 2`
2. Масштабирование размеров:
   - `height *= currentScale`
   - `diameter *= currentScale`
   - `radius *= currentScale`
3. Расчёт объёма (приближённый, методом цилиндрических слоёв):
   - Для каждой пары соседних точек профиля (r1, h1) и (r2, h2):
     - `segmentVolume = PI * ((r1² + r1*r2 + r2²) / 3) * (h2 - h1)`
     - `totalVolume += segmentVolume`
   - `totalVolume *= currentScale³`
4. Расчёт площади поверхности:
   - Для каждого треугольника geometry:
     - Вычислить площадь через векторное произведение: `0.5 * |(v2-v1) × (v3-v1)|`
     - `totalArea += triangleArea`
   - `totalArea *= currentScale²`
5. Расчёт количества треугольников:
   - `triangleCount = geometry.index ? geometry.index.count / 3 : geometry.attributes.position.count / 3`
6. Расчёт количества вершин:
   - `vertexCount = geometry.attributes.position.count / 3`
7. Обновление UI-элементов отображения размеров:
   - Обновить текстовые поля с текущими значениями
   - Форматирование: 2 знака после запятой для размеров, 0 для треугольников
8. Отображение визуальных индикаторов:
   - Если diameter > 220 мм: красный индикатор (не поместится на стандартную платформу)
   - Если height > 200 мм: жёлтый индикатор (высокая модель)
   - Если volume < 100 мм³: предупреждение (очень маленькая модель)

## Обработка ошибок

| Код ошибки | Ситуация | Действие системы | Сообщение пользователю |
|------------|----------|------------------|------------------------|
| ERR-931 | Нет geometry | Остановка на шаге 1 | `Нет данных для расчёта размеров.` |
| ERR-932 | Geometry повреждена | На шаге 1 | `Невозможно вычислить размеры: геометрия повреждена.` |
| ERR-933 | Infinity в расчётах | На шаге 3-4 | `Расчёт размеров остановлен: некорректные данные.` |
