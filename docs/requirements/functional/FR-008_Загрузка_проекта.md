## Описание

Система восстанавливает состояние проекта из сохранённых данных (локальное хранилище или импортированный JSON-файл) и отображает его в редакторе.

## Входящие данные

| Параметр | Тип | Описание | Обязательный |
|----------|-----|----------|--------------|
| projectId | string | ID загружаемого проекта | Нет (если импорт) |
| importFile | File | JSON-файл для импорта | Нет (если по ID) |
| hasUnsavedChanges | boolean | Есть ли несохранённые изменения | Нет, по умолчанию false |

## Выходные данные

| Параметр | Тип | Описание |
|----------|-----|----------|
| projectId | string | ID загруженного проекта |
| formulaText | string | Восстановленная формула |
| maxHeight | number | Восстановленная высота |
| segments | number | Восстановленное количество сегментов |
| cameraPosition | Vector3 | Восстановленная позиция камеры |
| cameraTarget | Vector3 | Восстановленный target камеры |
| scale | number | Восстановленный масштаб |
| orientation | string | Восстановленная ориентация |
| isSuccess | boolean | Результат загрузки |
| errorMessage | string | Сообщение об ошибке (если неудача) |

## Логика обработки

1. Обработка несохранённых изменений:
   - Если hasUnsavedChanges === true:
     - Показать диалог: `Есть несохранённые изменения. Сохранить? Да / Нет / Отмена`
     - Если "Отмена": вернуть `isSuccess: false, skip: true`
     - Если "Да": вызвать FR-007 (сохранение)
     - Если "Нет": продолжить
2. Загрузка из хранилища по ID:
   - `stored = localStorage.getItem('project_' + projectId)`
   - Или: запрос к IndexedDB по ID
3. Загрузка из JSON-файла:
   - `reader = new FileReader()`
   - `reader.onload = () => { parsed = JSON.parse(reader.result); }`
   - `reader.readAsText(importFile)`
4. Валидация загруженных данных:
   - Проверка наличия обязательных полей: formula, name
   - Проверка типов: maxHeight — number, segments — number
   - Проверка формата: createdAt — ISO string, updatedAt — ISO string
   - Если данные повреждены: показать ошибку
5. Восстановление состояния редактора:
   - Установить `formulaText` в поле ввода формулы
   - Установить `maxHeight` в поле высоты
   - Установить `segments`, `scale`, `orientation`
   - Установить позицию камеры
6. Перестройка 3D-модели:
   - Перезапустить парсер формулы (FR-001 + FR-002)
   - Сгенерировать геометрию (FR-003)
   - Обновить 3D-сцену (FR-004)
7. Обновление заголовка / названия проекта
8. Показ уведомления: `Проект '{name}' загружен`
9. Обновление UI: выделить загруженный проект в списке

## Обработка ошибок

| Код ошибки | Ситуация | Действие системы | Сообщение пользователю |
|------------|----------|------------------|------------------------|
| ERR-701 | Проект не найден | Остановка на шаге 2 | `Проект '{name}' не найден.` |
| ERR-702 | Повреждённые данные | На шаге 4 | `Данные проекта повреждены. Попытка восстановления...` |
| ERR-703 | Неверный формат JSON | На шаге 4 | `Файл имеет неверный формат. Ожидается JSON проекта.` |
| ERR-704 | Старая версия формата | На шаге 4 | `Формат проекта устарел. Попытка конвертации...` |
| ERR-705 | Формула невалидна | На шаге 5 | `Формула в проекте невалидна. Исправьте её в редакторе.` |
