## Описание

Система генерирует STL-файл (стереолитографии) из 3D-геометрии вазы в выбранном формате (бинарный или ASCII) и инициирует его скачивание в браузер пользователя.

## Входящие данные

| Параметр | Тип | Описание | Обязательный |
|----------|-----|----------|--------------|
| vertices | Float32Array | Массив вершин от FR-003 | Да |
| indices | Uint16Array | Массив индексов треугольников от FR-003 | Да |
| normals | Float32Array | Массив нормалей от FR-003 | Нет |
| format | string | `binary` или `ascii` | Нет, по умолчанию `binary` |
| filename | string | Имя файла для скачивания | Нет, по умолчанию `vase_export.stl` |
| scale | number | Масштабный множитель | Нет, по умолчанию 1.0 |

## Выходные данные

| Параметр | Тип | Описание |
|----------|-----|----------|
| blob | Blob | STL-файл для скачивания |
| fileSize | number | Размер файла в байтах |
| triangleCount | number | Количество треугольников |
| downloadUrl | string | URL для триггера скачивания |

## Логика обработки

1. Валидация входных данных:
   - vertices и indices должны быть non-null
   - indices.length должно быть кратно 3 (каждый треугольник = 3 вершины)
2. Проверка валидности треугольников:
   - Каждый индекс < vertices.length / 3
   - Нет дублирующихся вершин в треугольнике (v1 != v2, v2 != v3, v1 != v3)
3. Подготовка данных для экспорта:
   - Если scale != 1.0: умножить координаты всех вершин на scale
   - Если normals переданы: вычислить нормали для каждого треугольника
4. Генерация STL:
   - Бинарный формат:
     - 80 байт: заголовок (any 80 bytes)
     - 4 байта: количество треугольников (uint32 LE)
     - Для каждого треугольника (50 байт):
       - 12 байт: нормаль (float32 x, y, z, little-endian)
       - 36 байт: 3 вершины (float32 x, y, z × 3)
       - 2 байта: атрибут байт count (uint16, обычно 0)
   - ASCII формат:
     - `solid name\n`
     - Для каждого треугольника:
       - `  facet normal nx ny nz\n    outer loop\n`
       - `    vertex x1 y1 z1\n    vertex x2 y2 z2\n    vertex x3 y3 z3\n    endloop\n  endfacet\n`
     - `endsolid name\n`
5. Создание Blob: `new Blob([arrayBuffer], { type: 'application/octet-stream' })`
6. Создание URL: `downloadUrl = URL.createObjectURL(blob)`
7. Создание временной ссылки:
   - `a = document.createElement('a')`
   - `a.href = downloadUrl`
   - `a.download = filename`
   - `a.click()`
8. Очистка:
   - `URL.revokeObjectURL(downloadUrl)`
   - Удаление временной ссылки
9. Возврат результатов: blob, fileSize, triangleCount
10. Показ уведомления пользователю

## Обработка ошибок

| Код ошибки | Ситуация | Действие системы | Сообщение пользователю |
|------------|----------|------------------|------------------------|
| ERR-401 | Нет геометрии | Остановка на шаге 1 | `Нет геометрии для экспорта. Создайте модель.` |
| ERR-402 | Некорректные индексы | На шаге 2 | `Геометрия повреждена. Проверьте формулу.` |
| ERR-403 | Файл слишком большой (>100MB) | После шага 4 | `Файл превышает 100MB. Уменьшите детализацию.` |
| ERR-404 | Браузер не поддерживает Blob | На шаге 5 | `Ваш браузер не поддерживает скачивание файлов.` |
| ERR-405 | Пользователь отменил скачивание | На шаге 7 | `Скачивание отменено.` (тихо, без ошибки) |
