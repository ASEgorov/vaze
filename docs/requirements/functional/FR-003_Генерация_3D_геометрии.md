## Описание

Система создаёт треугольную сетку (mesh) 3D-модели вазы на основе вычисленных точек. Форма определяется формулой, которая может зависеть от высоты `h` и/или угловой координаты `theta` (0 → 2π). Геометрия генерируется через Custom BufferGeometry с поддержкой волнистых, риплёных и спиральных поверхностей.

## Входящие данные

| Параметр | Тип | Описание | Обязательный |
|----------|-----|----------|--------------|
| parsedExpression | AST | Дерево разбора формулы (от FR-001) | Да |
| segments | number | Количество сегментов по окружности (8-1024) | Нет, по умолчанию 64 |
| pointsCount | number | Количество точек по высоте (10-1000) | Нет, по умолчанию 100 |
| scale | number | Масштабный множитель (0.1-5.0) | Нет, по умолчанию 1.0 |
| maxHeight | number | Максимальная высота в мм (10-1000) | Да |

## Выходные данные

| Параметр | Тип | Описание |
|----------|-----|----------|
| geometry | BufferGeometry | Готовая 3D-геометрия с position, normal, uv атрибутами |
| triangleCount | number | Количество треугольников |
| vertexCount | number | Количество вершин |
| boundingBox | BoundingBox | Ограничивающий параллелепипед |
| volume | number | Оценочный объём модели (мм³) |
| surfaceArea | number | Оценочная площадь поверхности (мм²) |

## Логика обработки

1. Проверка входных данных: maxHeight в [10, 1000], segments в [8, 1024], pointsCount в [10, 1000], scale в [0.1, 5.0]
2. Вычисление шагов: `dTheta = 2*PI / segments`, `dH = maxHeight / pointsCount`
3. Генерация вершин (вложенный цикл):
   - Для j от 0 до pointsCount:
     - `h = j * dH`
     - Для i от 0 до segments:
       - `theta = i * dTheta`
       - Вычислить `r = evaluateFormula(parsedExpression, {h, theta})`
       - Если `r < 0`: зафиксировать предупреждение, использовать `r = 0`
       - Преобразование: `x = r * cos(theta) * scale`, `y = h`, `z = r * sin(theta) * scale`
       - Сохранить вершину: `positions.push(x, y, z)`
       - Сохранить UV: `uvs.push(i / segments, j / pointsCount)`
       - Сохранить начальную нормаль: `normals.push(cos(theta), 0, sin(theta))`
4. Генерация индексов (quad → 2 треугольника, CCW winding):
   - Для j от 0 до pointsCount-1:
     - Для i от 0 до segments-1:
       - `a = j * (segments + 1) + i`
       - `b = a + 1`
       - `c = (j + 1) * (segments + 1) + i`
       - `d = c + 1`
       - `indices.push(a, b, c)` (треугольник 1)
       - `indices.push(b, d, c)` (треугольник 2)
5. Создание BufferGeometry и установка атрибутов
6. Пересчёт нормалей: `geometry.computeVertexNormals()` для гладкого освещения
7. Расчёт bounding box: min/max по X, Y, Z
8. Расчёт объёма (приближённо через метод призмы между слоями)
9. Расчёт площади поверхности (сумма площадей треугольников по формуле Герона)
10. Возврат geometry, triangleCount, vertexCount, boundingBox, volume, surfaceArea

## Обработка ошибок

| Код ошибки | Ситуация | Действие системы | Сообщение пользователю |
|------------|----------|------------------|------------------------|
| ERR-201 | Меньше 2 точек | Остановка на шаге 1 | `Минимум 2 точки для генерации. Проверьте формулу.` |
| ERR-202 | segments вне диапазона | Остановка на шаге 1 | `Сегменты должны быть от 16 до 1024` |
| ERR-203 | scale вне диапазона | Остановка на шаге 1 | `Масштаб должен быть от 0.1 до 5.0` |
| ERR-204 | Все радиусы = 0 | После шава 2 | `Все радиусы равны 0. Ваза не имеет объёма.` |
| WARN-201 | Самопересечение | На шаге 7 | `Модель может содержать самопересечения.` |
