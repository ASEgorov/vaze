## Описание

Система предоставляет пользователю интерфейс для настройки параметров печати: масштаб модели, ориентация на платформе, единицы измерения и проверка совместимости с платформой принтера.

## Входящие данные

| Параметр | Тип | Описание | Обязательный |
|----------|-----|----------|--------------|
| scale | number | Масштабный множитель (0.10-5.00) | Нет, по умолчанию 1.0 |
| orientation | string | Ориентация: `base`, `side`, `custom` | Нет, по умолчанию `base` |
| units | string | Единицы измерения: `mm`, `cm`, `in` | Нет, по умолчанию `mm` |
| printerPlatform | object | Параметры платформы принтера | Нет |

## Выходные данные

| Параметр | Тип | Описание |
|----------|-----|----------|
| adjustedGeometry | BufferGeometry | Геометрия с применёнными настройками |
| dimensions | Dimensions | Размеры модели после настройки |
| fitsOnPlatform | boolean | Помещается ли на платформе |
| warningMessages | string[] | Предупреждения |

## Логика обработки

1. Валидация параметров:
   - scale в диапазоне [0.10, 5.00]
   - orientation из whitelist: base, side, custom
   - units из whitelist: mm, cm, in
2. Применение масштаба:
   - Для каждой вершины geometry:
     - `v.x *= scale`, `v.y *= scale`, `v.z *= scale`
   - Если units = 'cm': умножить на 0.1
   - Если units = 'in': умножить на 25.4
3. Применение ориентации:
   - base: без трансформации (основание на оси Z)
   - side: применить поворот на 90° вокруг оси X:
     - `newY = z`, `newZ = -y`
   - custom: применить матрицу вращения из углов Эйлера (x, y, z)
4. Расчёт bounding box после трансформации:
   - `minX = min(vertices.x)`, `maxX = max(vertices.x)`
   - `minY = min(vertices.y)`, `maxY = max(vertices.y)`
   - `minZ = min(vertices.z)`, `maxZ = max(vertices.z)`
   - `width = maxX - minX`
   - `depth = maxZ - minZ`
   - `height = maxY - minY`
5. Проверка платформ принтера:
   - Предустановленные платформы:
     - Small: 180×180 мм
     - Medium: 220×220 мм
     - Large: 300×300 мм
     - Custom: пользовательские размеры
   - Если `width > platformWidth` или `depth > platformDepth`:
     - `fitsOnPlatform = false`
     - Добавить предупреждение: `Модель не помещается на платформу ${platformWidth}×${platformDepth} мм`
6. Отображение размеров в реальном времени:
   - Обновить UI-элементы с текущими dimensions
   - Формат: `Высота: ${height.toFixed(1)} мм × Диаметр: ${Math.max(width, depth).toFixed(1)} мм`
7. Обновление 3D-сцены с adjustedGeometry:
   - Заменить геометрию на сцене
   - Обновить bounding box
8. Кэширование параметров для экспорта:
   - Сохранить в state: lastScale, lastOrientation
   - Использовать при экспорте в STL (FR-005)

## Обработка ошибок

| Код ошибки | Ситуация | Действие системы | Сообщение пользователю |
|------------|----------|------------------|------------------------|
| ERR-921 | scale вне диапазона | Остановка на шаге 1 | `Масштаб должен быть от 0.10 до 5.00` |
| ERR-922 | Неверная ориентация | На шаге 1 | `Неподдерживаемая ориентация.` |
| ERR-923 | Платформа не выбрана | На шаге 5 | `Выберите размер платформы или введите свои размеры.` |
| WARN-921 | Не помещается на платформу | На шаве 5 | `Модель не помещается на платформу ${w}×${d} мм` |
| WARN-922 | Очень маленькая модель | После шава 4 | `Модель меньше 5 мм в любом измерении.` |
