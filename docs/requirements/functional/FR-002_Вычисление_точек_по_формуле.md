## Описание

Система вычисляет значения радиуса сечения для заданного диапазона высоты и угловой координаты на основе парсеной формулы. Формула может зависеть от высоты `h` (0 → maxHeight) и/или угловой координаты `theta` (0 → 2π). Возвращает точки для генерации геометрии через BufferGeometry (см. FR-003).

## Входящие данные

| Параметр | Тип | Описание | Обязательный |
|----------|-----|----------|--------------|
| parsedExpression | AST | Дерево разбора формулы (от FR-001) | Да |
| maxHeight | number | Максимальная высота в мм (от 10 до 1000) | Да |
| segments | number | Количество точек по окружности (от 8 до 1024) | Нет, по умолчанию 64 |
| pointsCount | number | Количество точек по высоте (от 10 до 1000) | Нет, по умолчанию 100 |

## Выходные данные

| Параметр | Тип | Описание |
|----------|-----|----------|
| points3D | Point3D[][] | Двумерный массив точек [{r, h, theta}] для каждого сегмента |
| warnings | CalculationWarning[] | Предупреждения о NaN, отрицательных радиусах, разрывах |
| stats | CalculationStats | Статистика: minR, maxR, avgR, triangleCount, hasTheta |

## Логика обработки

1. Проверка входных данных: maxHeight в [10, 1000], segments в [8, 1024], pointsCount в [10, 1000]
2. Определение наличия theta: если формула содержит `theta`, устанавливаем `hasTheta = true`
3. Вычисление шагов: `dTheta = 2*PI / segments`, `dH = maxHeight / pointsCount`
4. Генерация точек (вложенный цикл):
   - Для j от 0 до pointsCount:
     - `h = j * dH`
     - Инициализировать ряд точек: `rowPoints = []`
     - Для i от 0 до segments:
       - `theta = i * dTheta`
       - Вычислить `r` путём подстановки `{h, theta}` в `parsedExpression`
       - Если результат NaN/Infinity: записать WARN-101, установить `r = 0`
       - Если `r < 0`: записать WARN-102, установить `r = 0`
       - Добавить точку `{r, h, theta}` в `rowPoints`
     - Добавить `rowPoints` в `points3D`
5. Проверка на разрывы (между соседними h при фиксированном theta):
   - Если `abs(r[j][i] - r[j-1][i]) > 2 * averageDelta`: записать WARN-103
6. Валидация: массив не должен быть пустым, количество точек >= 2 на ряд
7. Расчёт статистики: minR, maxR, avgR, minHeight, maxHeight, triangleCount = pointsCount * segments * 2, hasTheta
8. Возврат точек, предупреждений и статистики

## Обработка ошибок

| Код ошибки | Ситуация | Действие системы | Сообщение пользователю |
|------------|----------|------------------|------------------------|
| ERR-101 | Пустое AST | Остановка на шаге 1 | `Формула не скомпилирована. Проверьте синтаксис.` |
| ERR-102 | maxHeight вне диапазона | Остановка на шаге 1 | `Высота должна быть от 10 до 1000 мм` |
| ERR-103 | Результат NaN для всех точек | После шага 4 | `Формула не даёт корректных значений. Проверьте выражение.` |
| ERR-104 | Переполнение стека | Во время вычисления | `Вычисление заняло слишком много времени. Упростите формулу.` |
| WARN-101 | NaN/Infinity радиус | На шаге 4 | `Ошибка вычисления радиуса. Проверьте формулу.` |
| WARN-102 | Отрицательный радиус | На шаге 4 | `Радиус меньше 0. Точка исключена из модели.` |
| WARN-103 | Разрыв в профиле | На шаге 5 | `Резкий скачок радиуса при h=${h}. Модель может деформироваться.` |
