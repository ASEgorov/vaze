## Описание

Система сериализует текущее состояние проекта (формула, параметры, настройки камеры, камеры) в JSON-формат и сохраняет в локальное хранилище браузера (IndexedDB / localStorage).

## Входящие данные

| Параметр | Тип | Описание | Обязательный |
|----------|-----|----------|--------------|
| formulaText | string | Текстовая формула | Да |
| maxHeight | number | Максимальная высота (мм) | Да |
| segments | number | Количество сегментов | Нет, по умолчанию 64 |
| cameraPosition | Vector3 | Позиция камеры | Нет |
| cameraTarget | Vector3 | Target камеры | Нет |
| scale | number | Масштаб модели | Нет, по умолчанию 1.0 |
| orientation | string | Ориентация модели | Нет, по умолчанию `base` |
| projectName | string | Имя проекта | Нет, генерируется автоматически |

## Выходные данные

| Параметр | Тип | Описание |
|----------|-----|----------|
| projectId | string | Уникальный ID проекта (UUID) |
| projectData | object | Полная сериализованная структура |
| createdAt | string | ISO timestamp создания |
| updatedAt | string | ISO timestamp обновления |
| isSuccess | boolean | Результат сохранения |
| errorMessage | string | Сообщение об ошибке (если неудача) |

## Логика обработки

1. Валидация обязательных полей: formulaText не пустой, maxHeight в [10, 1000]
2. Проверка изменений: сравнение текущего состояния с последним сохранённым (deepEqual)
   - Если изменений нет: вернуть `isSuccess: true, skip: true`
3. Генерация имени проекта (если не указано):
   - Формат: `Ваза_YYYY-MM-DD_HH:mm`
   - Проверка уникальности в хранилище: если дубликат, добавить индекс `_#N`
4. Создание структуры проекта:
   ```
   {
     id: generateUUID(),
     name: projectName,
     formula: {
       text: formulaText,
       maxHeight: maxHeight,
       segments: segments,
       scale: scale,
       orientation: orientation
     },
     camera: {
       position: cameraPosition || defaultPosition,
       target: cameraTarget || defaultTarget
     },
     createdAt: now(),
     updatedAt: now()
   }
   ```
5. Проверка доступного места в хранилище:
   - Если используется localStorage: проверка оставшегося объёма (< 5MB)
   - Если используется IndexedDB: проверка квоты
6. Проверка лимита проектов:
   - Загрузить список существующих проектов
   - Если >= 50: предложить удалить старые или удалить самый старый автоматически
7. Запись в хранилище:
   - `localStorage.setItem('project_' + projectId, JSON.stringify(projectData))`
   - Или: `db.transaction('projects', 'readwrite').put(projectData)`
8. Обновление списка проектов в хранилище:
   - Загрузить `projectList`
   - Добавить projectId в начало
   - Сохранить `projectList`
9. Показ уведомления: `Проект '{name}' сохранён`
10. Возврат результатов

## Обработка ошибок

| Код ошибки | Ситуация | Действие системы | Сообщение пользователю |
|------------|----------|------------------|------------------------|
| ERR-601 | Пустая формула | Остановка на шаге 1 | `Нельзя сохранить: введите формулу.` |
| ERR-602 | maxHeight вне диапазона | На шаге 1 | `Высота должна быть от 10 до 1000 мм` |
| ERR-603 | Хранилище заполнено | На шаге 6 | `Хранилище заполнено. Удалите старые проекты.` |
| ERR-604 | IndexedDB недоступен | На шаге 7 | `Ошибка сохранения: хранилище недоступно.` |
| ERR-605 | Серверная синхронизация не доступна | При облачном сохранении | `Невозможно сохранить: нет подключения к интернету.` |
