# ADR-001: Выбор React Three Fiber вместо чистого Three.js

## Контекст

VaseForge — фронтенд-приложение на React. Для 3D-рендеринга есть два основных подхода:
1. Использовать чистый Three.js с императивным API
2. Использовать React Three Fiber (R3F) — declarative React-обёртку над Three.js

Необходимо выбрать подход, который обеспечит лучший баланс между удобством разработки, производительностью и поддерживаемостью.

## Решение

Использовать **React Three Fiber (R3F)** как основную библиотеку для 3D-рендеринга.

### Почему R3F:
- **Декларативный подход**: 3D-объекты описываются как React-компоненты, что интегрируется с JSX-экосистемой
- **Хуки React**: useFrame, useThree, useMemo — доступ ко всем хукам React для 3D-логики
- **DevTools**: R3F DevTools для отладки сцены, компонентов и props
- **Иммутабельность**: состояние сцены управляется через React state — предсказуемо и легко тестируемо
- **Компонентная модель**: каждая часть сцены (меш, свет, контроллеры) — изолированный компонент
- **Поддержка от сообщества**: R3F — стандарт де-факто для React+3D, регулярные обновления

### Компоненты экосистемы R3F:
- `@react-three/fiber` — ядро
- `@react-three/drei` — полезные хелперы (OrbitControls, Text, Environment и т.д.)
- `zustand` — управление глобальным состоянием (интегрируется с R3F через useStore)

## Последствия

### Положительные
- Быстрая разработка благодаря декларативному подходу
- Переиспользование компонентов (материалы, контролы, эффекты)
- Простая интеграция с UI-элементами React (панели, кнопки, инпуты)
- Типизация через TypeScript — полная поддержка из коробки
- Лёгкая тестирование компонентов через React Testing Library
- Drei предоставляет готовые решения для OrbitControls, GridHelper, AxesHelper

### Отрицательные
- Дополнительный overhead от React-абстракции (незначительный для наших сцен)
- Необходимость изучения R3F-конвенций (вместо чистого Three.js)
- Некоторые Three.js-фичи требуют «ручного» доступа к renderer через useThree
- Зависимость от экосистемы R3F (но она стабильна и поддерживается)

## Альтернативы

| Альтернатива | Плюсы | Минусы | Оценка |
|--------------|-------|--------|--------|
| Чистый Three.js | Полный контроль, меньше абстракций | Императивный код, сложная интеграция с React, нет DevTools | - |
| React Three Fiber + drei | Declarative, экосистема, DevTools | Зависимость от R3F, learning curve | + |
| Svelte Three | Лучшая производительность | Svelte не используется в стеке, меньше экосистема | - |
