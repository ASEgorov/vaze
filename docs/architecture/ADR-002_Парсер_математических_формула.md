# ADR-002: Выбор библиотеки для парсинга математических формул

## ID
ADR-002

## Статус
- [x] Предложено
- [ ] Принято
- [ ] Отклонено
- [ ] Устарело

## Контекст

Система принимает математические формулы от пользователей в текстовом формате (например, `r = 5 + 2 * sin(h * 0.1)`) и должна:
- Парсить выражение в AST (Abstract Syntax Tree)
- Валидировать синтаксис и семантику
- Вычислять значения для заданного диапазона
- Предоставлять понятные сообщения об ошибках
- Предотвращать выполнение вредоносного кода

Необходимо выбрать между готовой библиотекой (math.js) и кастомным парсером.

## Решение

Использовать **math.js** как основную библиотеку для парсинга и вычисления математических формул.

### Почему math.js:
- **Parser + Evaluator**: встроенный парсер выражений с AST и JIT-компиляцией
- **Function map**: whitelist-механизм — можно ограничить доступ только к нужным функциям (sin, cos, pow и т.д.)
- **Units**: встроенная поддержка единиц измерения (опционально, для FR-012)
- **Error messages**: детальные сообщения об ошибках с позициями
- **Performance**: JIT-компиляция, кэширование AST
- **TypeScript**: полная типизация из коробки
- **Мини-файл**: `math.js/es` — можно импортировать только нужное (parser, evaluate) — ~50 КБ gzipped
- **Безопасность**: evaluation в изолированном scope — нет доступа к window, document, globalThis

### Конфигурация безопасности:
```typescript
const math = mathjs.create(mathjs.all, {
  map: 'Array',
  memory: false,
  safety: true
});

// Whitelist функций
const allowedFunctions = ['sin', 'cos', 'tan', 'pow', 'abs', 'sqrt', 'log', 'exp'];
const scope = {
  h: 0,
  theta: 0,
  pi: Math.PI,
  e: Math.E
};

// Ограничение по времени
const result = math.evaluate(expr, scope, { timeout: 1000 });
```

## Последствия

### Положительные
- Готовый парсер с хорошей диагностикой ошибок
- Не нужно писать свой парсер — экономия недель разработки
- Поддержка сложных выражений: вложенные функции, константы, операторы
- Массовое вычисление: `math.evaluate(expr, { h: [0,1,2,...] })` — векторный режим
- Документация и сообщество
- TypeScript-типы

### Отрицательные
- Размер библиотеки: ~200 КБ ungzipped, ~50 КБ gzipped (только parser)
- Некоторые функции math.js выходят за рамки наших потребностей (матрицы, комплексные числа) — но их можно не импортировать
- JIT-компиляция может быть медленнее для очень простых выражений (не критично при <500 символов)

## Альтернативы

| Альтернатива | Плюсы | Минусы | Оценка |
|--------------|-------|--------|--------|
| math.js | Готовый парсер, whitelist, типизация, JIT | ~50 КБ gzipped | + (выбрано) |
| Кастомный парсер (recursive descent) | Минимальный size, полный контроль | Недели разработки, сложно обеспечить безопасность, нужно писать все ошибки | - |
| Function Constructor (new Function) | Быстрый | **НЕ БЕЗОПАСНО** — позволяет выполнять любой JS-код | ✗ |
| expr-eval | Лёгкий, простой | Меньше функций, хуже error messages, нет JIT | - |
