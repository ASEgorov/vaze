# ADR-004: Генерация 3D-геометрии — Custom BufferGeometry

## ID
ADR-004

## Статус
- [x] Предложено
- [x] Принято
- [ ] Отклонено
- [ ] Устарело

## Контекст

Вазы — объекты вращения. Форма задаётся формулой, которая может:
1. Зависеть только от высоты: `r = f(h)` — осесимметричная ваза
2. Зависеть от высоты и угла: `r = f(h, theta)` — волнистая, рипленная, спиральная форма

Необходимо выбрать подход к генерации геометрии, покрывающий оба случая.

## Решение

Использовать **Custom BufferGeometry** — единую логику генерации меша через `new THREE.BufferGeometry()`.

### Почему единый подход:
- **Одна кодовая база**: нет дублирования логики для двух случаев
- **Поддержка theta**: волны, риплы, спирали — без ограничений
- **Контроль атрибутов**: полный контроль над вершинами, нормалью, UV, цветами
- **Простота**: один путь от формулы до геометрии — проще тестировать и отлаживать
- **Масштабируемость**: легко добавить новые переменные (например, `time` для анимации)

### Алгоритм генерации

```typescript
function generateVaseGeometry(
  formula: string,
  params: { maxHeight: number; segments: number; pointsCount: number; scale: number }
): BufferGeometry {
  const positions: number[] = [];
  const normals: number[] = [];
  const uvs: number[] = [];
  const indices: number[] = [];

  const dTheta = (2 * Math.PI) / params.segments;
  const dH = params.maxHeight / params.pointsCount;

  for (let j = 0; j <= params.pointsCount; j++) {
    const h = j * dH;
    for (let i = 0; i <= params.segments; i++) {
      const theta = i * dTheta;

      // Вычисляем радиус с учётом h и theta
      const r = evaluateFormula(formula, { h, theta });
      const scaledR = r * params.scale;

      // Полярные координаты → декартовы
      const x = scaledR * Math.cos(theta);
      const z = scaledR * Math.sin(theta);
      const y = h; // ось Y — вертикаль

      positions.push(x, y, z);
      uvs.push(i / params.segments, j / params.pointsCount);

      // Приближённая норма: вектор из оси наружу
      const nx = Math.cos(theta);
      const nz = Math.sin(theta);
      normals.push(nx, 0, nz);
    }
  }

  // Генерация индексов (quad → 2 треугольника)
  for (let j = 0; j < params.pointsCount; j++) {
    for (let i = 0; i < params.segments; i++) {
      const a = j * (params.segments + 1) + i;
      const b = a + 1;
      const c = (j + 1) * (params.segments + 1) + i;
      const d = c + 1;

      // Counter-clockwise winding
      indices.push(a, b, c);
      indices.push(b, d, c);
    }
  }

  const geometry = new BufferGeometry();
  geometry.setAttribute('position', new Float32Array(positions));
  geometry.setAttribute('normal', new Float32Array(normals));
  geometry.setAttribute('uv', new Float32Array(uvs));
  geometry.setIndex(indices);
  geometry.computeVertexNormals(); // Пересчёт для гладкости
  return geometry;
}
```

### Параметры геометрии

| Параметр | Default | Диапазон | Назначение |
|----------|---------|----------|------------|
| pointsCount | 100 | 10–1000 | Точки по высоте |
| segments | 64 | 8–1024 | Сегменты по окружности |
| scale | 1.0 | 0.1–5.0 | Масштабный множитель |
| maxHeight | 150 | 10–1000 | Высота в мм |

### Поддерживаемые переменные в формуле

| Переменная | Диапазон | Описание | Пример использования |
|------------|----------|----------|---------------------|
| `h` | 0 → maxHeight | Высота в мм | `5 + 2 * sin(h * 0.1)` |
| `theta` | 0 → 2π | Угловая координата | `8 + 3 * cos(theta * 6)` |
| `pi` | — | Константа π | `pi`, `2 * pi` |
| `e` | — | Константа e | `e^2` |

### Примеры формул

```
# Осесимметричная ваза (theta игнорируется)
r = 5 + 2 * sin(h * 0.1)

# Волнистый край
r = 5 + 2 * sin(h * 0.1) + 1 * cos(theta * 8)

# Рипленная ваза (вертикальные полосы)
r = 5 + 1.5 * cos(theta * 6) * sin(h * 0.05) + 0.5

# Спиральная ваза
r = 5 + 1 * sin(theta + h * 0.2)

# Коралловая текстура
r = 4 + 2 * sin(h * 0.3) + 0.8 * cos(theta * 12 + h * 0.5)
```

### Поддерживаемые функции
Все функции math.js, разрешённые whitelist'ом (см. ADR-002):
`sin, cos, tan, asin, acos, atan, pow, sqrt, abs, log, exp, floor, ceil, round, min, max, mod`

## Последствия

### Положительные
- **Единая кодовая база**: одна реализация, проще тестировать
- **Поддержка theta**: волны, риплы, спирали, коралловые текстуры — всё работает
- **Полный контроль**: вершины, нормали, UV, индексы — всё под нашим контролем
- **Нормали**: `computeVertexNormals()` — гладкое освещение
- **UV**: правильная развёртка для UV-текстур (опционально)
- **Масштабируемость**: легко добавлять новые переменные и эффекты

### Отрицательные
- Нет нативной оптимизации LatheGeometry — чуть больше overhead (незаметно на современных устройствах)
- Нужно самостоятельно следить за winding order (CCW)
- Выше overhead при очень больших моделях (1000+ segments) — но для STL-экспорта это не проблема

## Альтернативы

| Альтернатива | Плюсы | Минусы | Оценка |
|--------------|-------|--------|--------|
| Custom BufferGeometry | Одна кодовая база, theta, полный контроль | Нет нативной LatheGeometry-оптимизации | + (выбрано) |
| LatheGeometry fallback | Оптимизация для r=f(h) | Нет theta, два пути, сложнее поддержка | - |
| Только LatheGeometry | Простота, нативная оптимизация | Нет волн, нет theta | ✗ |
| Shader displacement | Красиво, GPU-ускорение | Нет нормалей для STL, сложно с UV | ✗ |
