# VaseForge

## Описание

VaseForge — веб-приложение для создания 3D-моделей ваз в формате STL для 3D-печати. Пользователь задаёт параметры и формулы, на основе которых система автоматически строит геометрию вазы и позволяет экспортировать результат в STL-файл.

## Структура документации

```
docs/
├── requirements/
│   ├── business/        — Бизнес-требования (BR-XXX)
│   ├── functional/      — Функциональные требования (FR-XXX)
│   └── non-functional/  — Нефункциональные требования (NFR-XXX)
├── user-stories/        — User Stories (US-XXX)
├── use-cases/           — Use Cases (UC-XXX)
└── architecture/        — ADR, архитектурные решения
```

## Быстрый старт

1. **Ознакомьтесь с [`QWEN.md`](QWEN.md)** — конфигурация проекта, стек, правила
2. **Создайте документ** — скопируйте шаблон из `templates/` или соответствующей папки `docs/*/`
3. **Заполните ID** — используйте систему нумерации из QWEN.md
4. **Укажите связи** — заполните секцию «Связанные элементы»

## Система идентификаторов

| Тип | Префикс | Пример |
|-----|---------|--------|
| Бизнес-требование | BR | BR-001 |
| Функциональное требование | FR | FR-001 |
| Нефункциональное требование | NFR | NFR-001 |
| User Story | US | US-001 |
| Use Case | UC | UC-001 |
| Задача | TASK | TASK-001 |
| ADR | ADR | ADR-001 |

## Рабочий процесс

```
Бизнес-требования (BR)
    ↓
Функциональные требования (FR)
    ↓
Use Cases (UC) + User Stories (US)
    ↓
Задачи (TASK)
```

## Шаблоны

Все шаблоны находятся в:
- `docs/requirements/business/TEMPLATE.md` — бизнес-требование
- `docs/requirements/functional/TEMPLATE.md` — функциональное требование
- `docs/requirements/non-functional/TEMPLATE.md` — нефункциональное требование
- `docs/user-stories/TEMPLATE.md` — User Story
- `docs/use-cases/TEMPLATE.md` — Use Case
- `docs/architecture/TEMPLATE.md` — ADR
- `tasks/TEMPLATE.md` — задача
- `templates/` — дополнительные шаблоны
